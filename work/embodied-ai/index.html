This case answers

说说你如何把跨多个学科的论文库，做成零基础读者也能选路径、做对比、形成研究摘要的学习产品。

关键取舍是把“收录”和“已深度核验”分开，不让内容数量替代可信度说明。

On this case

问题系统关键取舍证据边界

156 篇论文不是入口，最多只是库存。

具身智能横跨视觉、语言、控制、世界模型、数据集、仿真、射频和听觉。把论文按时间堆在一起，会让零基础读者在术语出现之前就失去方向。

产品要解决的是“接下来读什么、为什么读、读完如何比较”：先用 22 章教程建立直觉，再按任务、30 天核心路线或主题三连进入论文。

一个由内容、数据 API 与全前端状态组成的静态产品。

  1. Markdown 笔记与解析材料进入构建脚本，生成统一的论文对象。
  2. 标签、主题、时代和 venue 被聚合为索引、对比表、时间线、图谱与热力图。
  3. 教程、30+5 学习路径、主题三连和阅读包提供不同进入方式。
  4. localStorage 记录已读、连续阅读、路径进度和下一篇推荐，并支持版本化导入导出。
  5. 构建后输出页面、Pagefind 搜索、JSON 数据 API、feed、sitemap 与 PWA 资源。
  • Node.js
  • marked
  • gray-matter
  • Pagefind
  • KaTeX
  • D3.js
  • localStorage
  • GitHub Pages

让内容产品既有规模，也有诚实的可信度边界。

从生活类比开始，但不止于类比

术语第一次出现必须有定义与日常直觉，公式后要有人话解释；同时保留原论文信息、实验解读与思考题，方便读者回到一手材料。

同一内容提供多种阅读视图

Timeline 回答“怎么演化”，Compare 回答“同主题方法怎么不同”，Graph 和 Heatmap 回答“哪些概念经常一起出现”。

个人进度留在用户设备

阅读状态和推荐逻辑完全在前端运行，不需要登录或上传学习数据；导入导出让用户保留控制权。

内容状态与质量边界。

Research boundary

156 篇笔记的 deep-read 状态表示满足结构与最低内容门禁，不等于作者已逐页人工复核每篇论文。内容由 AI 辅助整理，关键事实、数字和结论仍应回到原论文核验。

当前仓库说明中，46 篇保留本地解析文本与 SHA-256 清单，110 篇引用 HTTPS 原文页面。两种来源路径的可追溯强度不同，站点不再用统一“深读”标签掩盖差异。
